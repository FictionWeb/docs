Fiction uses most popular and pre-installed dependencies and has support for multiple cores for network. 

## Requirements 

- bash 4.0+ (4.* support in theory, not fully tested)
- GNU coreutils
- netcat/ncat/socat (optional, for HTTPS and fork support)
- awk, grep, sed (mostly preinstalled)

## Installation

:::note
By default, Fiction uses `socat` for connection.

If your machine doesn't include `socat`, change `core` option to bash in `config.json` or other supported core, or install socat with your system's package manager if possible.
   
```diff lang="json" title="config.json"
-  "core": "socat",
+  "core": "bash",
```
:::

<Steps>

1. ### Install Fiction by cloning the github directory

   ```sh
   git clone https://github.com/FictionWeb/framework
   ```

2. ### Go to the framework directory 

    ```sh 
    cd framework
    ```

3. ### Make `fiction` executable binary

   ```bash
   ln -s $(pwd)/fiction.so.sh /usr/bin/fiction && chmod +x /usr/bin/fiction
   ```

4. ### Start the server
   <Tabs>
   <TabItem label="fiction">
   ```sh 
   fiction dev
   ```
   </TabItem>
   <TabItem label="bash">
   You can Fiction's main file directly as well
   ```sh 
   bash fiction.so.sh dev
   ```
   </TabItem>
   </Tabs>

5. ### Check if the server is working

   Open `<server IP>:8080` in your browser; If you see the default page, Fiction is working and you can start editing the code in `index.shx` file in `pages` folder.

</Steps>
